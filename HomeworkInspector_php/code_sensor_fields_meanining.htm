I_refs

Number of instructions consumed. A smaller value means faster code.

 

check_pattern

The ratio of the number of passed checks divided by the total number of checks. 1.0 means all checks are passed.

There are a total of 10 checks for HW2. The checks will check if the address returned by acquire () is valid (i.e. on the pool) and also check if there’s any corruption in the data stored in the memory blocks.

 

gcc

This is the error code from compiling your code with g++. The value is 0 if the compilation completes without any error.

 

illegal_func (i.f.)

If you use any disallowed system call / library function, this will be non-zero.

 

illegal_header (i.h.)

If you include any disallowed header file, this will be non-zero.

 

m_peak

The “peak” memory usage in bytes. This includes all memory usage (m_heap + m_heap_extra + m_stack + m_text + m_data + m_bss).

 

Note this records the “peak” usage during the execution of the program, not the memory usage at program exit.

 

m_heap

The size of heap memory in use at the time when m_peak is sampled.

 

A 0 only means that at the time when mem_total is at its peak, the heap memory usage is 0 byte. It does not necessarily mean your code uses no heap memory at all.

 

Any direct or indirect call to malloc / new consumes heap memory.

 

m_heap_extra (m_heap_e)

Space used by the heap bookkeeping structure (the “free-list” maintained by the C library).  

 

heap_lost

If you forgot to free / delete an allocated block, the size of that “lost” block goes in here.

 

m_stack

The stack usage at the time when m_peak is sampled.

 

Stack memory is used to house local variables and the call arguments and return pointers used in function calls.

 

m_data, m_bss

http://en.wikipedia.org/wiki/Data_segment

 

m_text

http://en.wikipedia.org/wiki/Text_segment

 

m_error (m.e.)

The number of invalid memory access (e.g. reading from uninitialized area, writing to protected area, etc.)

 

m_total

The summation of memory usage over the lifetime of the program under test (i.e. the area under the curve in the following figure).

missing

The number of fields without sensor readings. This should be 0 unless your code terminates unexpectedly during the analysis process.
